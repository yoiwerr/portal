# 如何导入正确格式？

ChatLab 要求聊天记录使用固定格式才能正确解析。

## 格式规范

每行一条消息，格式为：

```
[发送者 时间]: 消息内容
```

### 规则

| 规则 | 说明 |
|------|------|
| 发送者 | 人名或昵称，如 `张三`、`我`、`对方` |
| 时间 | 任意格式均可，如 `2024-06-01 10:30`、`10:30`、`06-01 22:00` |
| 分隔符 | 英文冒号 `:` 或中文冒号 `：` 均可 |
| 消息内容 | 该条消息的完整文本 |
| 每行一条 | 一条消息占一行，不能跨行 |

### 示例

```
[张三 2024-06-01 10:30]: 你好，今天有空一起吃个饭吗？
[我 2024-06-01 10:32]: 好啊，几点？
[张三 2024-06-01 10:33]: 十二点怎么样？老地方
[我 2024-06-01 10:35]: 没问题，到时候见
[张三 2024-06-01 10:36]: 好的，不见不散
```

### 支持的导入方式

- **文件上传**：将格式化的 `.txt`、`.md` 或 `.json` 文件拖拽上传（不支持截图、图片等二进制格式）
- **手动粘贴**：直接粘贴到输入框
- **JSON 格式**：也可使用 `[{"sender":"张三","timestamp":"10:30","content":"你好"}]` 数组结构

> 如果你手头是截图或文档，请参照下方「万能提示词」让 AI 帮你转换。

## 万能提示词：让 AI 帮你格式化

如果你手头有原始聊天记录（微信截图文字、QQ 导出、其他格式），把下面的提示词发给 **任意大模型**（ChatGPT、DeepSeek、通义千问等），它会把你的聊天记录转换成 ChatLab 可读的格式。

直接复制以下内容发给 AI：

---

```
请将我提供的聊天记录转换为以下固定格式，每行一条消息：

[发送者名称 时间]: 消息内容

规则：
1. 方括号内的「发送者」和「时间」之间用空格分隔
2. 方括号后面跟英文冒号或中文冒号
3. 冒号后有一个空格，然后是消息正文
4. 每条消息占一行，不要有空行
5. 时间格式保持一致，如都是 "HH:MM" 或都是 "YYYY-MM-DD HH:MM"
6. 如果原记录中时间不精确，可以只保留 "时:分" 部分
7. 对方的名字请保持统一（不要一会儿"张三"一会儿"小张"）
8. 忽略原记录中的系统消息（如"你撤回了一条消息"、"对方正在输入..."等）
9. 只输出格式化后的文本，不要加任何解释或代码块标记

以下是需要转换的聊天记录：

[在这里粘贴你的原始聊天记录]
```

### 使用步骤

1. 复制上面的提示词
2. 打开 ChatGPT / DeepSeek / 通义千问 / 任意大模型
3. 粘贴提示词，把最后一行的 `[在这里粘贴你的原始聊天记录]` 替换为你的真实数据
4. 发送，拿到 AI 格式化后的结果
5. 将结果粘贴到 ChatLab 的「手动输入」框中（或保存为 `.txt` 文件上传）
6. 点击「情感分析」或「气氛分析」即可

### 常见场景

| 你的原始数据 | 怎么做 |
|-------------|--------|
| 微信聊天截图 | 先用微信自带的「复制文字」或手机长按选文字，粘贴出来再用提示词格式化 |
| QQ 聊天记录 | QQ 导出为 txt 后用提示词转换 |
| 短信截图 | 手动抄写或用手机 OCR 提取文字，再用提示词格式化 |
| 其他 App 聊天 | 截图或用 App 自带导出功能提取文字，再用提示词格式化 |
